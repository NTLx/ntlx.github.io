# AI Agent 时代的开源贡献问题——Hugging Face 用一份 Skill 教 AI 学会「不写」

## 核心论点
1. AI Agent 淹没开源：PR 量暴增十倍，维护者审查带宽没涨
2. 不是代码的问题，是理解的问题：Agent 缺的不是写代码能力，是理解代码库的能力
3. Skill 的本质是「不」字诀：15000 字大部分是约束，教 Agent 学会说不
4. 审查者也是用户：开源瓶颈不是代码生成，是注意力
5. 独立测试：不信任 Agent 的自我评估，设计验证机制

## 关键数据
- 3000万→10亿程序员（Jensen Huang）
- cURL 漏赏计划：6年、8.6万美元、因 AI slop 终止
- 16小时内收到7份 AI 生成垃圾漏洞报告
- 92% 开发者已用 AI 编程工具
- 45% AI 生成代码未通过安全测试
- Claude Code 贡献了 49 个 CVE（11 个严重级别）
- Skill 包含近 15000 字
- transformers 是模型定义的"事实来源"

## Skill 约束要点（「不」字诀）
- 不改共享工具——除非维护者明确同意
- 不加注释解释代码——审查者要同时读注释和代码更累
- 不提重构——没有被要求的改进就是添乱
- 不过早抽象——自包含的模型文件比精巧的继承结构好维护
- 不接受任何想法——维护者会 push back，你也要学会 push back
- PR 必须披露是 agent 辅助的
- 不要把审查者意见直接丢回 Agent 处理
