## 简介
本文件面向 NTLx's Blog 的发布自动化技能模块，系统性梳理以下三大技能：
- **wechat-article-write**：微信公众号文章写作流水线（15步完整流程，端到端整合多技能，含图片上传与发布）
- **baoyu-post-to-wechat**：微信公众号文章发布（支持 API 与浏览器两种方式，含多账号与权限检查）
- **github-image-hosting**：GitHub 图床上传（jsDelivr CDN）

文档覆盖发布流程全链路、API 集成方法、权限与多账号机制、浏览器自动化、错误处理策略、性能优化与故障排查，并提供微信公众号开发的配置指南与调试方法。

## 项目结构
发布自动化技能位于 .agents/skills 目录下，采用"技能-脚本"组织方式：
- 技能目录包含 SKILL.md（技能说明）与 scripts/（可执行脚本）
- 各脚本通过 Bun 运行，依赖 Node.js 生态与第三方库
- wechat-article-write 作为编排技能，串联多个 baoyu-*、github-image-hosting、web-access 等技能

## 核心组件
- **wechat-article-write**
  - **15步完整流程**：资料收集 → 文章创作 → 质量门控 → 分类确认 → 封面/信息图/插图并行 → 统一格式检测 → 图床前置上传 → 占位符替换 → 去 AI 痕迹 → Markdown 格式化 → HTML 转换（本地文件） → 博客发布轨（先发布） → 等待Pages部署（可选） → 微信发布轨（后发布）
  - **状态管理系统**：state.mjs（命令行工具）+ state-lib.mjs（库函数），提供统一的状态读写和断点续跑能力
  - **双产物机制简化**：同时生成article.md（博客CDN版）和article-local.md（微信本地路径版），但微信轨全程使用本地文件，零CDN依赖
  - **关键门控**：质量门控、引用验证、格式统一检测、微信轨验证
- **baoyu-post-to-wechat**
  - API 发布：wechat-api.ts（令牌获取、正文图片上传、素材上传、草稿发布）
  - 浏览器发布：wechat-browser.ts（CDP 控制 Chrome，自动填写标题/内容/图片，保存草稿）
  - 配置与多账号：wechat-extend-config.ts（EXTEND.md 解析、账户解析、凭证加载）
  - Markdown 渲染：md-to-wechat.ts（占位符替换、主题渲染、HTML 输出）
  - 图片处理：wechat-image-processor.ts（格式检测、尺寸压缩、编码转换）
- **github-image-hosting**
  - 上传脚本：upload.ts（gh CLI 调用、冲突检测、jsDelivr CDN 返回）

## 详细组件分析

### wechat-article-write：微信文章写作流水线（15步完整流程）

**更新** 微信文章写作管线现已升级为15步完整流程，移除了CDN回退机制，简化了双产物机制。

#### 15步完整流程
- **Step 0**：依赖预检（硬门控）→ 检查各技能安装状态和npm依赖
- **Step 1**：资料收集（三级降级）→ 通过web-access技能获取URL内容
- **Step 2**：文章创作（Skill工具调用）→ ljg-writes生成初稿
- **Step 2.4.1**：质量门控（硬门控）→ 字数、互动、引用、数据点检查
- **Step 2.5**：分类确认（硬门控）→ 分类推荐+用户确认+frontmatter写入
- **Step 3+4+4.5**：封面图+插图+信息图并行生成
- **Step 3.4.2**：统一格式检测（硬门控）→ JPEG-PNG格式修正
- **Step 4.6**：图床前置上传（硬门控）→ 一次性上传所有图片并等待CDN传播
- **Step 5**：占位符→CDN URL（硬门控）→ 生成article.md（博客CDN版）和article-local.md（微信本地路径版）
- **Step 6**：去AI痕迹（硬门控）→ humanizer-zh检测与修正
- **Step 7**：Markdown格式化 → baoyu-format-markdown排版优化
- **Step 8**：HTML转换（本地文件）→ 生成微信专用HTML（article-wechat.html）
- **Step 9**：博客发布轨（先发布）→ publish-blog.mjs同步到博客项目
- **Step 9.5**：等待Pages部署（可选）→ wait-pages-deployed.mjs轮询验证
- **Step 10**：微信发布轨（后发布）→ publish-wechat.mjs发布到公众号草稿

#### 状态管理系统
- **state.mjs**：命令行工具，提供init/get/set/next/dump等状态操作
- **state-lib.mjs**：库函数，供所有Step脚本导入使用，避免重复spawn
- **统一状态写入**：大部分步骤都内置了状态写入，agent只需调用对应脚本

#### 双产物机制简化
- **同时生成两个文件**：article.md（博客CDN版）和article-local.md（微信本地路径版）
- **微信轨专用**：article-local.md用于微信轨，包含本地imgs/路径引用
- **博客轨专用**：article.md用于博客轨，包含CDN URL
- **零CDN依赖**：微信轨全程使用本地文件，wechat-api.ts直接读取本地图片上传

### baoyu-post-to-wechat：微信公众号文章发布
- **发布方式**
  - API：适合快速、稳定发布，需 AppID/AppSecret；支持正文图片与封面上传、草稿新增
  - 浏览器：适合无需 API 凭证的场景，通过 CDP 自动化填写与保存
- **权限与配置**
  - EXTEND.md 支持全局与多账号配置，含默认主题、颜色、评论开关、Chrome 个人资料路径
  - 凭证加载顺序：EXTEND.md 账户字段 → 环境变量 → 项目级 .env → 用户级 .env
- **图片处理**
  - 正文图片上传前进行格式检测与必要转换，确保不超过 1MB 且为微信支持格式
  - 封面使用素材接口上传，正文图片使用正文图片接口返回 URL
- **工作流要点**
  - 不要预转换 Markdown 为 HTML；由脚本内部处理占位符与渲染
  - 默认将普通外链转为底部引用，可通过参数关闭
  - **零CDN依赖**：微信轨全程使用本地文件，wechat-api.ts直接读取本地图片上传

## 性能考虑
- **15步完整流程优化**
  - Step 3（封面）、Step 4（插图）、Step 4.5（信息图）三者并行执行，显著缩短总耗时
  - Step 4.6 图床前置上传，避免后续步骤重复上传，减少网络开销
- **双产物机制优化**
  - 同时生成article.md和article-local.md，避免重复占位符替换
  - 微信轨全程使用本地文件，避免CDN传输延迟
- **状态管理优化**
  - state-lib.mjs避免重复spawn，提高状态写入效率
  - 断点续跑支持，减少重复执行
- **图片处理优化**
  - API 模式下正文图片自动压缩与格式转换，避免超限与不兼容
  - 统一格式检测在 Step 3.4.2 前执行，减少遗漏与重复处理
- **浏览器自动化优化**
  - CDP 直连用户 Chrome，避免重复登录与抓取失败
- **博客发布优化**
  - 先博客后微信的发布顺序，确保sourceUrl的稳定性

## 故障排查指南
- **常见问题与修复**
  - 缺少 API 凭证：按引导设置 AppID/AppSecret，或在 EXTEND.md/环境变量中配置
  - 访问令牌错误：检查凭证有效性与有效期
  - 未登录（浏览器模式）：首次运行会打开浏览器，扫码登录后继续
  - Chrome 未找到：设置 WECHAT_BROWSER_CHROME_PATH
  - 标题/摘要缺失：使用自动生成或手动提供
  - 无封面：添加 frontmatter cover 或放置 imgs/cover.png
  - 评论默认值不符：检查 EXTEND.md 中 need_open_comment/only_fans_can_comment
  - 粘贴失败：检查系统剪贴板权限
- **15步流程特定问题**
  - Step 2.5 分类确认失败：检查分类推荐准确性，必要时手动选择
  - Step 4.6 图床上传失败：检查GitHub token和仓库权限，重试上传
  - Step 5 占位符替换失败：检查image-map.json完整性，确认占位符与图片编号对齐
  - Step 9 博客发布失败：检查frontmatter字段映射，确保构建验证通过
  - Step 10 微信发布失败：检查sourceUrl可达性，确认封面和HTML文件存在
- **状态管理问题**
  - 状态写入失败：检查state-lib.mjs权限，确认posts目录可写
  - 断点续跑异常：使用state.mjs next查看当前步骤状态
- **双产物问题**
  - article.md仍有本地路径：检查Step 5占位符替换是否正确执行
  - article-local.md路径不正确：检查imgs/目录结构和文件命名
  - HTML转换失败：检查article-local.md格式和主题配置
