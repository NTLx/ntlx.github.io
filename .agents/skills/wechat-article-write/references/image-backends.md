# 图片后端构图特性

记录各图片生成后端的构图倾向和已知限制，减少重试轮次。

## Google Gemini (gemini-3.1-flash-image-preview 等)

**构图倾向**：
- 强烈倾向于**居中构图**，无论 aspect ratio 设为什么
- 16:9 格式下，视觉内容通常集中在中间约 5:4 区域，两侧留大片暗色空白
- 仅靠 prompt engineering（如 "MUST fill entire frame"）无法解决此问题

**解决方案**：
- 选择**天然横跨宽画幅**的构图概念：水平分割、左右对比、全景场景
- 避免：单一主体居中、冰山隐喻、垂直堆叠的概念图
- 如果必须用居中主体，接受两侧留白作为设计元素（暗色背景 + 居中光效）

**何时选择 Gemini**：
- 1:1 或 4:3 格式（居中构图不是问题）
- 水平分割/对比类构图（天然填满宽度）
- 需要快速出图时（Gemini 通常比 GPT Image 快）

**已知问题**：
- 格式声明可能与实际不符（如声明 PNG 但实际是 JPEG）
- 发布脚本会自动处理格式不匹配

## OpenAI GPT Image 2

**构图倾向**：
- 对 aspect ratio 的遵循比 Gemini 好
- 16:9 格式下能较好地填充画面
- 但 16:9 仍可能比 Gemini 慢

**何时选择 GPT Image 2**：
- 需要精准构图（居中主体 + 16:9 格式）
- 当 Gemini 多次重试仍无法填满画幅时
- 封面图、对构图要求高的场景

**限制**：
- 需要 `OPENAI_API_KEY`
- 速度比 Gemini 慢

## DashScope (阿里通义万象)

**构图倾向**：
- 对中文 prompt 理解较好
- 构图风格偏写实/插画风

**何时选择**：
- 需要中国审美风格的插图
- 中文提示词复杂度高时

## Seedream (豆包) / Jimeng (即梦)

**构图倾向**：
- 偏向中国审美，适合公众号文章风格
- 对中文 prompt 支持好

**何时选择**：
- 需要更"中国风"或"公众号风格"的配图
- 面向国内读者的文章封面

## 构图概念速查

当用户反馈"图片有空白边"或"构图不对"时：

| 用户反馈 | 可能原因 | 优先解决方案 |
|---------|---------|-------------|
| "右半边空白" | Gemini 居中构图 | 改为水平分割/左右对比概念 |
| "内容太集中" | 任何后端的居中倾向 | 改为全景/平铺式构图 |
| "画面太空" | 概念本身内容不足 | 增加具体元素描述 |
| "风格不对" | 后端风格不匹配 | 切换后端（国内审美 → DashScope/Seedream） |

## 重试策略

1. **第一次重试**：优化 prompt（加构图约束，如 "MUST fill entire 16:9 frame"）
2. **第二次重试**：如果还是同样问题，**改变构图概念**而非继续优化 prompt
   - 居中主体 → 水平分割
   - 冰山隐喻 → 左右对比
   - 垂直堆叠 → 平铺全景
3. **第三次重试**：考虑切换后端（如有 OPENAI_API_KEY → GPT Image 2）

**核心原则**：如果构图问题是后端固有行为（如 Gemini 居中），换 prompt 没用——换概念或换后端。
